Mosaic System

Unifying how a portfolio of consumer apps looks, behaves, and scales — one shared design system, tightly coupled to the code it ships with.

Year
2024
Role
Product Designer
Duration
Ongoing
Tools
Figma, Tokens Studio, Storybook

The challenge

Lorem ipsum dolor sit amet. Every app had its own version of the same button, the same list row, the same paywall. Teams were re-inventing components in parallel, and polish was wildly inconsistent between products even when they shipped the same feature.

We needed a shared library that was opinionated enough to bring real consistency, and flexible enough that no team would feel it slowed them down.

Approach

Mosaic was built from a small token layer and a handful of core primitives first. We resisted the urge to ship every component on day one — instead we matched the library to the real components the apps were already leaning on, and grew it in lockstep with product work.

Figma variables and design tokens were generated from the same source so engineering and design stayed in sync automatically. Changing a radius in Figma changed it in production by Friday.

Outcome

Mosaic is now the default starting point for new screens across the portfolio. Teams ship faster, fewer one-off components get built, and the visual layer finally gets the same iteration rigor the codebase does.
