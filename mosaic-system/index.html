Mosaic System

A scalable design and tokens library that turned monetization across 100+ apps in the Bending Spoons portfolio into one orchestrated system — designed and shipped before the AI era, when "doing it at scale" still meant inventing the method yourself.

Year
2023 — 2024
Role
Co-architect · Product Designer
Scale
100+ apps in the portfolio
Tools
Figma, Tokenique, GitHub

The challenge

When we started, there was no scalable way to design monetization across the portfolio. Every app had its own paywall, its own onboarding, its own tokens. Every experiment was a one-off. Every visual update was a copy-paste job. Every learning was trapped inside the app it came from.

The brief was huge: build a system that could scale seamlessly to 100+ apps, make it possible to isolate the reason behind each experiment's success or failure, and keep brand identity intact across wildly different products — from fitness to meditation to photo editing.

And the harder part: nobody had done this before at our scale. We searched for case studies, internal precedents, off-the- shelf approaches — and found nothing. We had to invent the model.

How we built it

I co-architected Mosaic with one other designer. We shaped every layer of the system together — the architecture, the naming conventions, the modes, the sync with code — and then took it all the way to production across the full portfolio. We split the apps between us so each of us could go deep on real product cases, but the system itself was something we built side by side, problem after problem.

It grew into four connected layers, each one solving a problem the previous layer had exposed.

We started with primitives — a flat color palette with consistent naming, used as the raw building blocks but never applied to components directly. On top of those, we layered semantic tokens: meaningful names like button-primary-background or accent-color that referenced the primitives but carried intent instead of values. This is where the system started to feel like a system — and where the real leverage lived.

Each token operates on two axes at once. Six brand theme modes — pink, blue, green, purple, orange, turquoise — one for each app family, so switching theme repaints an entire screen automatically. And light and dark modes, paired with every token, so the same component lives in both worlds with zero duplicate work. A single component definition produces twelve valid variants out of the box.

The third layer is components, connected to tokens through Figma's variable modes. Change a theme on a frame and the change ripples through every nested component, popup, and flow. The system was built to be dev-ready by default: every token maps to a CSS variable, and code uses those tokens — not raw hex values.

The last layer closes the loop. Tokens are synced to GitHub through Tokenique. Designers update tokens in Figma, engineering pulls them straight into the codebase. The chronic drift between design and code became a non-issue.

What made it hard

Building this before AI tooling meant solving everything with manual rigor. We had to match naming conventions across thousands of variables, fight the limits Figma had at the time on modes and references, and find the right level of abstraction for every token. Too generic and they lose meaning. Too specific and they stop scaling. The line between the two is thinner than it looks.

We hit Figma's ceiling more than once. Re-architected the variable structure twice. Invented our own opacity-naming convention because the platform didn't support one yet. It was a research project disguised as a design system, and a lot of what eventually shipped came out of iterations we had to throw away.

The hardest decision, in the end, wasn't visual or technical — it was deciding what the system should not try to do, so that the parts it did do, it did really well.

Outcome

Mosaic became the engine behind monetization across 100+ apps. It supported 245 experiments that contributed to over $100M in profit. It gave design and engineering teams a shared vocabulary — and turned shipping a new paywall variant from a multi-week task into something that takes hours.

And the bigger, quieter win sits inside the company: Mosaic has officially become the onboarding system for most new designers joining Bending Spoons. The way we built it is now the way the company teaches scale.

What I learned

Mosaic changed how I think about scale. The hardest part wasn't visual or technical — it was knowing where to draw the line, and trusting that a smaller, sharper system would outperform a bigger, fuzzier one.

I'd build it the same way again. The piece I'd add: documenting the failed iterations early. Half of what made Mosaic work were the things we tried and abandoned, and that knowledge is exactly what new joiners need most.
